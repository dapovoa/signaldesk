## Python for automation and integration
### where-i-used-it
I used Python in real work for ETL, automation, reporting, validation, backend API work, and SAP-related integrations. It was one of the main tools I used when manual work was starting to repeat too much.

### real-use
I usually use Python when I need something practical, quick to change, and easy to keep under control. That includes integrations, reporting jobs, validation steps, and operational tooling.

### limits
I do not force Python everywhere. If the environment points to another stack, I follow the constraint instead of the language.
